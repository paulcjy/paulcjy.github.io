b:T8eb,<p><a href="https://github.com/vercel/next.js/tree/canary/examples/github-pages">Next.js 예제</a></p>
<ol>
<li>GitHub Repository를 생성한다.</li>
<li>Next.js 프로젝트를 생성한다.</li>
<li><code>.gitignore</code>에서 <code>/out/</code> 부분을 제거한다.</li>
<li><code>next.config.js</code>를 수정한다. <a href="#nextconfigjs-%EC%88%98%EC%A0%95">이동</a></li>
<li><code>package.json</code>에 <code>deploy</code> 스크립트를 추가한다. <a href="#deploy-%EC%8A%A4%ED%81%AC%EB%A6%BD%ED%8A%B8">이동</a></li>
<li>코드를 <code>main</code> 브랜치에 push 한다.</li>
<li><code>deploy</code> 스크립트를 실행하여 <code>gh-pages</code> 브랜치를 생성하고 빌드된 코드를 push 한다.</li>
<li>변경사항을 적용하고 싶을 때 <code>deploy</code> 스크립트를 실행한다.</li>
</ol>
<h1>next.config.js 수정</h1>
<pre><code class="language-ts">/** @type {import('next').NextConfig} */
const nextConfig = {
  output: 'export',
  basePath: '/&#x3C;리포지토리 이름>',
}

module.exports = nextConfig
</code></pre>
<p><code>output</code>은 <code>'export'</code>로 설정한다.</p>
<p><code>basePath</code>는 리포지토리 이름이 <em>username</em>.github.io라면 추가하지 않아도 된다.
아닐 때에만 리포지토리 이름을 적어준다.</p>
<h1>deploy 스크립트</h1>
<pre><code class="language-json">{
	"scripts": {
		...
		"deploy": "next build &#x26;&#x26; touch out/.nojekyll &#x26;&#x26; git add out/ &#x26;&#x26; git commit -m \"Deploy\" &#x26;&#x26; git subtree push --prefix out origin gh-pages"
	},
	...
}
</code></pre>
<p>다음 스크립트를 <code>deploy</code>로 설정한다.</p>
<pre><code>next build &#x26;&#x26; touch out/.nojekyll &#x26;&#x26; git add out/ &#x26;&#x26; git commit -m \"Deploy\" &#x26;&#x26; git subtree push --prefix out origin gh-pages
</code></pre>
<h1>주의</h1>
<p>위 방법으로 배포할 때는 <code>next.config.js</code>에 반드시 <code>output</code> 옵션이 있어야 한다. 그러나 <code>output</code> 옵션을 넣으면 <code>npm run dev</code>로 개발 서버를 실행할 때, <code>generateStaticParams()</code>로 dynamic route segments를 정적으로 생성하는 데 문제가 있다.</p>
