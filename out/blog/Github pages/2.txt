b:T1f90,<h1>1. Jekyll이란?</h1>
<blockquote>
<p><a href="http://jekyllrb-ko.github.io/">Jekyll 사이트</a></p>
</blockquote>
<p><strong>Jekyll</strong>은 정적 웹사이트 생성기이다. HTML이나 Markdown 등의 언어로 글을 작성하면 테마를 적용하여 웹 페이지를 만들어준다. 테마도 이미 만들어진 것 중에 선택할 수 있고, 내가 원하는 대로 다양한 설정을 만질 수 있다.</p>
<blockquote>
<p>어떤 글에서는 Jekyll이 Ruby로 만들어진 것이라 Ruby를 설치해야 한다고 했는데, 지금까지 블로그를 만들면서 한 번도 Ruby를 사용한 적이 없다. 테마의 파일을 그대로 받아와 사용하기 때문에 아직까지는 Ruby 없이도 충분히 가능할 것 같다. 로컬 환경에서 Jekyll을 더 활용하기 위해 필요하다고 한다.</p>
</blockquote>
<blockquote>
<p>(추가) 로컬 환경에서 페이지 미리보기를 하기 위해 필요했다.<br>
<a href="/github%20pages/07-local/">로컬에서 Jekyll 페이지 미리보기</a> 참고</p>
</blockquote>
<hr>
<h1>2. Jekyll 테마 선택</h1>
<p>테마는 굉장히 많다. 무료 테마도 있지만 유료인 것도 있다고 한다. 구글에 'jekyll 테마'라고 검색해도 많이 나온다. 테마 모음 사이트가 있다고 해서 몇 개 알아보았다.</p>
<ul>
<li><a href="http://jekyllthemes.org/">http://jekyllthemes.org/</a></li>
<li><a href="http://themes.jekyllrc.org/">http://themes.jekyllrc.org/</a></li>
<li><a href="https://jekyllthemes.io/">https://jekyllthemes.io/</a></li>
<li><a href="https://github.com/topics/jekyll-theme">https://github.com/topics/jekyll-theme</a></li>
<li><a href="https://jamstackthemes.dev/ssg/jekyll/">https://jamstackthemes.dev/ssg/jekyll/</a></li>
</ul>
<p>나는 **<a href="https://github.com/mmistakes/minimal-mistakes">minimal mistakes</a>**라는 테마를 선택했다. Github 블로그에서 많이들 사용하는 테마라고 한다. 심플하면서도 가볍고 빠르다고 해서 선택했다.</p>
<p><img src="https://user-images.githubusercontent.com/86853786/129292723-0a3a9dee-b400-4781-ab5d-09d500238a7c.png" alt="minimal mistakes skins"></p>
<p>한 테마 안에도 다양한 스킨이 있다. 물론 설정을 수정하면 디자인을 원하는 대로 바꿀 수 있다.</p>
<hr>
<h1>3. Jekyll 테마 가져오기</h1>
<p>테마를 repository에 올려야 테마가 적용된 페이지를 볼 수 있다. 지금 내 상황은 같은 <code>git</code>이 Github와 로컬 저장소, 두 군데에 있는 상황이다. 하나를 수정하면 다른 하나도 동기화를 시켜줘야 한다.</p>
<h2>1) 내 repository로 fork 하기</h2>
<blockquote>
<p>이 방법은 repository를 만들어서 clone 하기 전에 하는 것이 좋다. 더 편하게 repository를 만들 수 있다.</p>
</blockquote>
<p>테마를 선택하면 해당 테마의 Github 페이지로 연결되는 링크가 있을 것이다. 그 링크로 들어가서 오른쪽 상단의 <code>fork</code> 버튼을 누른다. <code>fork</code>는 다른 사람의 repository를 내 계정으로 복사하는 기능이다.</p>
<p><img src="https://user-images.githubusercontent.com/86853786/129294087-b5ddb445-38b4-4347-bb56-1ab1f90bd57e.png" alt="fork"></p>
<p><code>fork</code>가 끝나면 내 계정에 같은 이름으로 repository가 생성된 것을 볼 수 있다.</p>
<p><img src="https://user-images.githubusercontent.com/86853786/129294489-5c6b91fe-3b34-4658-a40e-5916845e977f.png" alt="내 repository"></p>
<p>Github pages를 이용하려면 repository의 이름이 <strong><em>username</em>.github.io</strong> 이어야 하므로 repository의 이름을 바꿔준다.</p>
<p><img src="https://user-images.githubusercontent.com/86853786/129294898-b07e8eb1-6550-45eb-ba9f-328278be462f.png" alt="image"></p>
<p>완성된 모습이다. 주소창에 <strong><em>username</em>.github.io</strong>를 치면 내 블로그가 뜬다.</p>
<p><img src="https://user-images.githubusercontent.com/86853786/129295179-049b00e2-0078-49e4-b004-7b8289deab08.png" alt="image"></p>
<p>그러고 나서 내 컴퓨터로 <code>clone</code> 하면 금방 테마를 받아올 수 있다.</p>
<h2>2) 내 컴퓨터에 저장하고 push 하기</h2>
<p>해당 테마의 Github 페이지로 들어가서 초록색 <code>code</code> 버튼을 누르고 <code>Download ZIP</code>을 클릭한다.</p>
<p><img src="https://user-images.githubusercontent.com/86853786/129295325-ea0f4a4b-cb10-46fd-adc0-d1d92f46faaa.png" alt="image"></p>
<p>다운받은 파일의 압축을 풀고 내용물을 전부 복사해서 앞에서 <code>clone</code> 했던 폴더에 붙여넣는다.</p>
<p><img src="https://user-images.githubusercontent.com/86853786/129295562-0dad71b5-1e78-4937-a89a-0a8cfeb7993b.png" alt="image"></p>
<p>마지막으로 이 명령들을 실행하여 내 컴퓨터에 있는 파일들을 Github로 보낸다.</p>
<pre><code>> git add .
> git commit -m "남기고 싶은 메시지"
> git push
</code></pre>
<p>앞으로도 내 컴퓨터에서 뭔가 수정을 하면 위의 명령어 3개를 실행해서 Github로 보낼 수 있다.</p>
<hr>
<h1>4. Jekyll 디렉토리 및 기본적인 설정들</h1>
<p>나도 Jekyll에 대해 잘 모르지만, 블로그를 만들면서 알게 된 내용들 위주로 적어보았다. <code>fork</code> 해 온 파일중에 <code>README.md</code>를 읽어보면 해당 테마에 대해 조금 이해할 수 있다.</p>
<p>또한, 블로그를 처음 만들면 블로그에 들어갔을 때 맨 위 오른쪽에 <a href="https://mmistakes.github.io/minimal-mistakes/docs/quick-start-guide/">Quick-Start Guide</a>라는게 있는데, 여기에 들어가면 유용한 정보를 많이 얻을 수 있다.</p>
<h2>디렉토리 및 파일</h2>
<p><code>_config.yml</code>은 설정 정보가 들어있는 파일이다. 수정을 많이 하게 된다.</p>
<p><code>_posts</code> 폴더는 블로그에 글을 올리는 폴더이다. 처음에는 없으므로 따로 생성해 주어야 한다. markdown 파일을 작성해서 이 폴더에 넣어주면 블로그에 글이 생긴다.</p>
<p><code>_pages</code> 폴더는 포스트가 아닌 페이지들을 담는 폴더이다. 이것도 처음에는 없고 따로 생성해 주어야 한다. markdown 파일로 페이지를 만들 수 있다.</p>
<p><code>_layout</code> 폴더는 페이지의 레이아웃을 보관하는 폴더이다. 나중에 글을 쓸 때 레이아웃을 지정하면 이 폴더에 있는 HTML 파일이 적용된다.</p>
<p><code>_sass</code> 폴더는 css 파일을 보관하는 폴더이다. 디자인을 수정하고 싶을 때 필요한 파일을 찾아서 수정한다.</p>
<p>이 외에도 <a href="https://mmistakes.github.io/minimal-mistakes/docs/structure/">Quick-Start Guide</a>에서 자세한 정보를 볼 수 있다.</p>
<h2>불필요한 파일 정리</h2>
