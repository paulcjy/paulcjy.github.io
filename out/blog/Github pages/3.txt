b:Tfd0,<p>이번에는 간단하게 글을 작성해보겠다.
포스트를 띄울 때 필요한 것은 Markdown 문법으로 작성한 본문과 Front Matter이다.
본문은 말 그대로 포스트에 나오는 내용이다.
Front Matter는 글에 대한 정보를 저장하는 부분이다.
메타데이터라고 생각하면 된다.</p>
<p></p>
<h1>1. Front Matter 작성하기</h1>
<p><strong>Front Matter</strong>는 파일 시작 부분에 작성하는 YAML 구역이다. 여기에는 글의 제목, 간단한 설명, 작성일, 분류, 태그 등의 정보를 저장할 수 있고, 심지어 변수까지도 저장해놓고 글에서 불러와 사용할 수 있다.</p>
<p>Front Matter를 작성할 때는 시작과 끝을 <code>---</code>로 감싸준다. 포스트를 작성할 때 기본적으로 사용하는 것들은 <code>title</code>, <code>excerpt</code>, <code>date</code>, <code>categories</code>, <code>tags</code> 등이 있다.</p>
<pre><code>---
title: "제목"
excerpt: "설명"
date: 2021-08-20

categories:
  - Github 블로그

tags:
  - [Github, Pages, blog, post, front matter, markdown]
---
</code></pre>
<ul>
<li><code>title</code>: 글의 제목이다.</li>
<li><code>excerpt</code>: 글에 대한 설명이다. 글 목록에서 제목 아래에 미리보기처럼 나온다.</li>
<li><code>date</code>: 글 작성일이다.</li>
<li><code>categories</code>: 카테고리를 설정할 수 있다. 나중에 카테고리별로 글을 모을 수 있다.</li>
<li><code>tags</code>: 태그이다. 마찬가지로 태그별로 글을 모을 수 있다.</li>
</ul>
<p>카테고리나 태그는 여러 개를 입력할 수 있는데, <code>-</code>를 사용하여 나열할 수도 있고, 배열처럼 대괄호로 묶어주어도 된다.</p>
<p>이외에도 다양한 Front Matter 설정값들이 있다. 나중에 더 소개하겠다.</p>
<hr>
<h1>2. 본문 작성하기</h1>
<p>본문은 워드문서 작성하듯이 쓰면 되지만 <strong>마크다운 문법</strong>이 적용된다. 몇 가지만 알아두면 쉽게 작성할 수 있다.</p>
<blockquote>
<p>마크다운 문법에 관한 자세한 내용은 <a href="/github%20pages/04-markdown/">(4) Markdown 문법</a>에서 볼 수 있다.</p>
</blockquote>
<p><strong>헤더</strong>와 <strong>코드블럭</strong>만을 사용해서 간단하게 글을 만들었다.</p>
<pre><code># Hello world 출력하는 방법

파이썬으로 "Hello world"를 출력할 떄는 이렇게 한다.

```
print('Hello world')
```
</code></pre>
<hr>
<h1>3. 글 업로드</h1>
<p>파일에 변경사항이 생기면 Github에 올려야 적용이 된다. 글을 새로 작성했을 때에도 마찬가지이다.</p>
<p>방법은 <code>git</code>에 <code>commit</code>을 하고 원격저장소(Github)에 <code>push</code> 하면 된다.</p>
<pre><code>> git add .
> git commit -m "남기고 싶은 메시지"
> git push
</code></pre>
<p>이 명령어 3개는 업로드할 때마다 실행시켜야 한다.</p>
<hr>
