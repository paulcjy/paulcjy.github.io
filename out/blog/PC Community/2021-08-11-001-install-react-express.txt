b:T1170,<h1>1. Node.js 설치</h1>
<p>React와 Express를 사용하기 위해서는 Node.js가 설치되어 있어야 한다.
Node.js란 JavaScript runtime이다.
원래 JavaScript는 웹 브라우저에서 작동하는 언어이지만, Node.js는 브라우저 밖에서도 JavaScript 코드를 실행할 수 있게 한다.</p>
<p><a href="https://nodejs.org">Node.js 다운로드</a></p>
<pre><code>> node -v
</code></pre>
<pre><code>> npm -v
v.14.17.3
</code></pre>
<h1>2. React와 Create React App</h1>
<p>React는 페이스북에서 만든 프론트엔드 프레임워크이다.
페이지의 모든 부분을 component 단위로 관리하고, SPA(Single Page Application)를 만들어낸다.
SPA란 페이지는 하나이지만 사용자가 어떤 동작을 하면 화면이 바뀌는 것이다.
이것은 새 페이지로 이동한 것이 아니다.
React는 추가로 필요한 정보만 서버에 요청하여 변화가 이루어져야 하는 부분만 수정한다.
</p>
<h2>Create React App</h2>
<p>React를 설치하기 전에 Create React App을 먼저 설치할 것이다.</p>
<pre><code>> npm install -g create-react-app
</code></pre>
<p>여기서 npm은 Node Package Manager로 Node.js에서 패키지를 설치하게 해준다.
Node.js를 설치할 때 자동으로 함께 설치되고 <code>npm install &#x3C;package></code> 형태로 실행한다.
<code>-g</code>는 전역 설치 옵션이다.
전역으로 설치한 패키지는 어디서든 사용할 수 있다.
<code>-g</code> 옵션을 쓰지 않으면 지역으로 설치되고, 설치한 디렉토리에서만 사용할 수 있다.</p>
<pre><code>> create-react-app .
</code></pre>
<p>위 명령은 현재 디렉토리에 React App을 생성한다.</p>
<pre><code>> create-react-app &#x3C;path>
</code></pre>
<p>위 명령은 해당 경로에 React App을 생성한다.
주의점은 앱이 설치되는 폴더 이름에는 대문자가 들어갈 수 없다.</p>
<h2>React</h2>
<p>이제 설치한 앱 폴더에 React를 지역 설치할 것이다.</p>
<pre><code>> npm install --save react react-dom
</code></pre>
<p>react와 react-dom, 2개의 패키지를 설치한다.
지역 설치를 하면 프로젝트 폴더 안에 <code>node_modules</code> 디렉토리가 생성되고 그 안에 패키지가 설치된다.
<code>--save</code> 옵션은 <code>package.json</code> 파일의 dependency 항목에 패키지를 추가한다.
참고로 npm5부터는 <code>--save</code> 옵션을 사용하지 않아도 자동으로 dependency 항목에 패키지가 추가된다.</p>
<h1>3. React 실행</h1>
<p>프로젝트 폴더 안에서 다음 명령어를 실행하면 React 서버가 시작된다.</p>
<pre><code>> npm start
</code></pre>

<h1>4. Express와 Express Generator</h1>
<p>Express와 Express Generator의 관계도 React와 Create React App의 관계와 같다.
Express는 백엔드 프레임워크이고, Express Generator는 기본 구성을 갖춘 Express 프로젝트를 간단하게 생성해준다.</p>
<p>마찬가지로 express-generator는 전역 설치를 하고, Express는 지역 설치를 한다.</p>
<pre><code>> npm install -g express-generator
</code></pre>
<pre><code>> express .
</code></pre>
